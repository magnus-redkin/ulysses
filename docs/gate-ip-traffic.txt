# Настройка multi-gate маршрутизации через SOCKS5
# Архитектура: Клиент → Гейт (HAProxy + send-proxy) → HFM (Xray) → SOCKS5 → Гейт → Интернет
# Дата актуализации: 22.07.2026

---

## Архитектура

Три независимых гейта, каждый со своей страной выхода:

G-1 (Финляндия): 62.60.249.53, 109.120.184.174, 83.147.216.201
G-2 (Швеция):    138.124.25.80
G-3 (Москва):    45.151.102.125, 77.221.151.75

HFM (ядро):      45.131.215.185

Трафик клиента заходит на гейт, HAProxy пробрасывает на HFM через WireGuard-туннель
с PROXY Protocol. HFM (Xray) маршрутизирует трафик обратно через SOCKS5 на тот же гейт,
с которого пришёл клиент. Гейт выпускает трафик в интернет через NAT.

Схема:
Клиент → G-X:443 (HAProxy send-proxy) → WG → HFM:4430X (Xray inbound gate-gX)
  → Xray routing (inbound gate-gX → outbound socks-gX)
  → SOCKS5 через WG → G-X:1080 (dante-server) → NAT → Интернет

WireGuard сеть: 10.90.0.0/16
HFM в WireGuard: 10.90.0.1
G-1 в WireGuard: 10.90.0.3
G-2 в WireGuard: 10.90.0.4
G-3 в WireGuard: 10.90.0.5

IP-алиасы на HFM (hiddifywg):
10.90.0.11 → G-1
10.90.0.12 → G-2
10.90.0.13 → G-3

Порты Xray на HFM:
44301 → G-1
44302 → G-2
44303 → G-3

---

## Часть 1: Гейт (на примере G-1, для G-2/G-3 аналогично)

### 1. HAProxy с send-proxy

Файл: /etc/haproxy/haproxy.cfg

frontend https_in
    bind *:443
    default_backend https_to_heart

backend https_to_heart
    server heart 10.90.0.11:44301 maxconn 25000 send-proxy
    # G-2: server heart 10.90.0.12:44302 maxconn 25000 send-proxy
    # G-3: server heart 10.90.0.13:44303 maxconn 25000 send-proxy

### 2. WireGuard (туннель к HFM)

Установка: sudo apt install -y wireguard

Файл: /etc/wireguard/wg0.conf

[Interface]
Address = 10.90.0.3/32
PrivateKey = 8MOBhOuqGgfIuhOLDtzr344cvcuJODF7hdub2V6kBWw=

[Peer]
PublicKey = zypHFdXUZrM10Zdc9gO+da0uEhxwHdQWyQ03qt76wl8=
Endpoint = 45.131.215.185:33712
AllowedIPs = 10.90.0.0/16
PersistentKeepalive = 25

Запуск:
sudo wg-quick up wg0
sudo systemctl enable wg-quick@wg0

### 3. NAT и форвардинг

sudo sysctl -w net.ipv4.ip_forward=1
echo "net.ipv4.ip_forward=1" | sudo tee -a /etc/sysctl.conf
sudo iptables -t nat -A POSTROUTING -s 10.90.0.0/16 -o enp0s3 -j MASQUERADE
sudo iptables -A FORWARD -i wg0 -j ACCEPT
sudo iptables -A FORWARD -o wg0 -j ACCEPT

Сохранить iptables:
sudo apt install -y iptables-persistent
sudo netfilter-persistent save

### 4. SOCKS5 прокси (dante-server)

Установка: sudo apt install -y dante-server

Файл: /etc/danted.conf

logoutput: syslog
internal: 0.0.0.0 port = 1080
external: enp0s3
socksmethod: none
clientmethod: none
user.privileged: root
user.unprivileged: nobody
client pass {
    from: 0.0.0.0/0 to: 0.0.0.0/0
    log: connect disconnect
}
socks pass {
    from: 0.0.0.0/0 to: 0.0.0.0/0
    log: connect disconnect
}

Запуск:
sudo systemctl restart danted
sudo systemctl enable danted

## Часть 2: HFM (Hiddify Manager)

### 1. WireGuard (туннель ко всем гейтам)

Файл: /etc/wireguard/hiddifywg.conf

[Interface]
Address = 10.90.0.1/16,fd42:42:90::1/90
ListenPort = 33712
PrivateKey = WFppp/dyCZp/O8LMFKxnkpD9qU2VDdgSP04/ieIXD3o=
Table = off

PostUp = iptables -I INPUT -p udp --dport 33712 -j ACCEPT
PostUp = iptables -I FORWARD -i enp0s3 -o hiddifywg -j ACCEPT
PostUp = iptables -I FORWARD -i hiddifywg -j ACCEPT
PostUp = iptables -t nat -A POSTROUTING -o enp0s3 -j MASQUERADE
PostUp = ip6tables -I FORWARD -i hiddifywg -j ACCEPT
PostUp = ip6tables -t nat -A POSTROUTING -o enp0s3 -j MASQUERADE

### 2. IP-алиасы на hiddifywg

При ребуте сбрасываются. Добавить в /etc/rc.local или crontab @reboot:

ip addr add 10.90.0.11/32 dev hiddifywg
ip addr add 10.90.0.12/32 dev hiddifywg
ip addr add 10.90.0.13/32 dev hiddifywg

### 3. Xray — inbound для каждого гейта

Файлы в /opt/hiddify-manager/xray/configs/:

05_inbounds_g1.json — inbound gate-g1 на 10.90.0.11:44301
05_inbounds_g2.json — inbound gate-g2 на 10.90.0.12:44302
05_inbounds_g3.json — inbound gate-g3 на 10.90.0.13:44303

Все с "acceptProxyProtocol": true, "security": "reality",
общие ключи Reality, path "/TZe1DA5Xmdguu8htyuGgnt",
sni "dl.google.com".

### 4. Xray — SOCKS5 outbound

Файл: /opt/hiddify-manager/xray/configs/06_outbounds_socks.json

Три outbound с тегами socks-g1, socks-g2, socks-g3,
каждый указывает на SOCKS5 проксю своего гейта:
- socks-g1 → 10.90.0.3:1080
- socks-g2 → 10.90.0.4:1080
- socks-g3 → 10.90.0.5:1080

### 5. Xray — routing

Файл: /opt/hiddify-manager/xray/configs/03_routing_gates.json

Правила:
- inbound gate-g1 → outbound socks-g1
- inbound gate-g2 → outbound socks-g2
- inbound gate-g3 → outbound socks-g3

### 6. Перезапуск Xray

sudo systemctl restart hiddify-xray
sudo systemctl status hiddify-xray

## Проверка

На HFM:
  curl --socks5 10.90.0.3:1080 ifconfig.me  # Должен показать IP G-1
  curl --socks5 10.90.0.4:1080 ifconfig.me  # Должен показать IP G-2
  curl --socks5 10.90.0.5:1080 ifconfig.me  # Должен показать IP G-3

На гейтах:
  nc -zv 10.90.0.11 44301  # G-1 → HFM
  nc -zv 10.90.0.12 44302  # G-2 → HFM
  nc -zv 10.90.0.13 44303  # G-3 → HFM

Клиентом: подключиться к каждой vless-ссылке, проверить IP на 2ip.ru.

## Восстановление после ребута

Гейты:
  - WireGuard: sudo wg-quick up wg0 (автозапуск через systemctl enable wg-quick@wg0)
  - HAProxy: автозапуск (systemctl enable haproxy)
  - Dante: автозапуск (systemctl enable danted)
  - iptables: сохранены через netfilter-persistent

HFM:
  - WireGuard: sudo wg-quick up hiddifywg
  - IP-алиасы: добавить в /etc/rc.local
  - Xray: автозапуск (hiddify-xray.service)

##### HAProxy
sudo apt install -y haproxy

sudo tee /etc/haproxy/haproxy.cfg << 'EOF'
global
    log /dev/log local0
    chroot /var/lib/haproxy
    stats socket /run/haproxy/admin.sock mode 660 level admin
    stats timeout 30s
    user haproxy
    group haproxy
    daemon
    maxconn 50000

defaults
    log     global
    mode    tcp
    option  tcplog
    option  dontlognull
    retries 3
    timeout connect 5s
    timeout client  2h
    timeout server  2h

frontend https_in
    bind *:443
    default_backend https_to_heart

backend https_to_heart
    server heart 10.90.0.11:44301 maxconn 25000 send-proxy
EOF

sudo systemctl restart haproxy
sudo systemctl enable haproxy

##### SOCKS5 (Dante)
sudo apt install -y dante-server

sudo tee /etc/danted.conf << 'EOF'
logoutput: syslog
internal: 0.0.0.0 port = 1080
external: enp0s3
socksmethod: none
clientmethod: none
user.privileged: root
user.unprivileged: nobody
client pass {
    from: 0.0.0.0/0 to: 0.0.0.0/0
    log: connect disconnect
}
socks pass {
    from: 0.0.0.0/0 to: 0.0.0.0/0
    log: connect disconnect
}
EOF

sudo systemctl restart danted
sudo systemctl enable danted

sudo journalctl -u danted -f

sudo tcpdump -i hiddifywg port 44301 -nn

sudo ss -tnp | grep -E "44301|44302|44303"

#### Проверка правил iptables на HFM:
sudo iptables -L INPUT -v -n
sudo iptables -t nat -L POSTROUTING -v -n
