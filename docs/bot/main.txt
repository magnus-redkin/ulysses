АРХИТЕКТУРА И ЛОГИКА РАБОТЫ КЛИЕНТСКОГО ТЕЛЕГРАМ-БОТА ULYSSES VPN

Данный модуль спроектирован по принципу «Тонкого клиента» (Thin Client) и полностью очищен от прямой бизнес-логики, проверок лимитов или прямых запросов к базе данных. Его единственная обязанность — рендеринг графического интерфейса мессенджера, перехват событий пользователя и отправка сигналов на центральный бэкенд FastAPI.

1. ВЕДОМАЯ СТЕЙТ-МАШИНА (STATE MACHINE)
Бот не хранит состояния шагов пользователя локально в памяти. При любом действии (ввод команды /start или клик по инлайн-кнопке) бот выполняет асинхронный HTTP-запрос к бэкенду. Центральная система вычисляет контекст и возвращает строго типизированный JSON-пакет:
- state: текущий статус шага (например, "payment_free").
- message: готовый, очищенный текст для вывода на экран.
- keyboard: кодовое имя клавиатуры, которую нужно отобразить.

Бот считывает имя клавиатуры, находит соответствующий генератор кнопок в локальном словаре KEYBOARDS и обновляет интерфейс в чате. Это позволяет централизовать логику переключения экранов меню на бэкенде. Изменение структуры меню больше не требует обновления и перезапуска самого бота.

2. ДИНАМИЧЕСКОЕ ПОСТРОЕНИЕ ИНТЕРФЕЙСА ТАРИФОВ
Клавиатура выбора тарифных планов не захардкожена в коде приложения. Когда бэкенд запрашивает вывод экрана "tariffs", бот выполняет дополнительный GET-запрос к API бухгалтерского учета, забирает актуальный конфигурационный массив цен и сроков, после чего динамически строит сетку синих кнопок для мессенджера.

3. АРХИТЕКТУРНАЯ ЗАЩИТА ОТ КРИТИЧЕСКОЙ ОШИБКИ «PARSE FAILED»
В процессе разработки была локализована и устранена уязвимость, приводившая к тому, что официальные приложения Telegram (особенно Desktop-версии) аварийно завершали рендеринг сообщений, оставляя пустой экран. Защита реализована на двух уровнях:
- Перевод на HTML-режим. Глобальный режим разметки бота переключен с Markdown на HTML. Markdown-символы (звездочки, дефисы, круглые скобки) интерпретировались клиентскими приложениями как битые элементы форматирования, если в названиях тарифов присутствовали спецсимволы. HTML полностью игнорирует эти знаки.
- Ограничение объема callback_data. Telegram API жестко лимитирует размер скрытых данных инлайн-кнопок (не более 64 байт). Использование длинных текстовых слагов совместно с тяжелыми эмодзи переполняло буфер пакета, вызывая сбой на стороне смартфона или ПК. Решение: генератор кнопок переведен на экстремально короткие префиксы (например, "t_free"), занимающие минимум байт и гарантирующие 100% отображение интерфейса на любых устройствах.

4. СЕТЕВАЯ СТАБИЛЬНОСТЬ И ИЗОЛЯЦИЯ ОШИБКАМИ
Взаимодействие с API бэкенда FastAPI реализовано через асинхронный клиент httpx. Если центральный сервер уходит на перезагрузку, испытывает проблемы с базой данных или возвращает статус-код ошибки, бот перехватывает исключение на уровне ядра, гасит аварийный Traceback и выводит пользователю безопасное системное уведомление «⚠️ Сервис временно недоступен». Это предотвращает падение процесса поллинга (aiogram) и сохраняет работоспособность интерфейса для других команд.
